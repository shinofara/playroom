# Heal - Healmate活動最適化ツール群

## プロジェクト概要

Healmate（ヒールメイト）での活動をAIパワーで最大化するためのツール群。
対象サービス: https://healmate.jp/

### Healmateとは
- 既婚者専用マッチングサービス（登録者50万人超）
- 男女比: 男性54% / 女性46%（7割が30〜40代）
- ブラウザのみ（ネイティブアプリなし）
- 主要機能: 会員検索、いいね、マッチング、メッセージ、掲示板、イベント
- 運営: レゾンデートル株式会社

### 料金体系（男性）
- スタンダード: 9,800円/月〜3,800円/月（長期割引あり）
- プレミアム: 11,800円/月〜5,800円/月

### プレミアム限定機能
- メッセージ付きいいね、グループチャット、ログイン状態確認
- イベント企画・参加、既読/未読確認、通話/ビデオ通話

## アーキテクチャ

### 技術スタック
- **ブラウザ操作**: Playwright MCP（CDP経由で既存Chrome接続）
- **AI分析/戦略**: Claude Code
- **データ蓄積**: `data/` 配下にJSON形式で保存

### ディレクトリ構成
```
heal/
├── CLAUDE.md          # プロジェクト設定
├── .mcp.json          # Playwright MCP設定
├── data/              # 収集データ（JSON）
│   ├── profiles/      # 閲覧した相手のプロフィール
│   ├── likes/         # いいね履歴
│   ├── messages/      # メッセージ履歴
│   └── my-profile/    # 自分のプロフィール変更履歴
├── scripts/           # 分析・レポート用スクリプト
├── .claude/
│   ├── skills/        # Claude Codeスキル定義
│   │   ├── like-judge.md       # いいね自動判断
│   │   ├── profile-fetch.md    # プロフィール取得
│   │   ├── search-research.md  # 検索リサーチ
│   │   ├── message-strategy.md # 初メッセージ戦略
│   │   └── message-analyze.md  # メッセージ分析
│   └── knowledge/     # スキル横断ナレッジ（永続化）
│       ├── persona_and_values.md  # ペルソナ・価値観・戦略方針
│       ├── user_preferences.md    # ターゲット評価基準・スコアリング
│       └── target_profiles.md     # ターゲット情報蓄積
└── .mcp.json
```

### Knowledge（スキル横断ナレッジ）
- `.claude/knowledge/persona_and_values.md` - ユーザーの価値観、求める関係性、対外的な人物像、メッセージ/プロフィール/写真の戦略方針。**全スキルの判断の核**。
- `.claude/knowledge/user_preferences.md` - お気に入り24人+価値観から構築した評価基準・スコアリングロジック。
- `.claude/knowledge/target_profiles.md` - いいねを返した/送った/マッチングした相手の情報。全スキルから参照・更新される。

### Playwright MCP利用方針
- Playwright MCPがChromiumブラウザを起動（CDPなし）
- Healmateにログイン済みの状態で操作する
- 操作は人間の行動を模倣する速度で行い、サーバーに過度な負荷をかけない

### 機能領域
| # | 領域 | 概要 |
|---|------|------|
| 1 | プロフィール運用 | 自己紹介文の改善、閲覧数・いいね率の変化追跡（SEO的PDCA） |
| 2 | いいね分析 | 送受信履歴、マッチ率、傾向分析 |
| 3 | メッセージ分析 | 返信率、会話パターン、勝ちパターン抽出 |
| 4 | 検索リサーチ | 条件別検索、候補のスコアリング |
| 5 | 戦略立案・初メッセージ | 相手プロフィールに合わせたアプローチ文面生成・送信 |

## 基本方針

- コメントやターミナルへの出力は日本語とする
- 基本的にソフトウェアの原理原則を前提とする
- 可能な限りFunctional Programingを前提とする
- 利用規約を遵守し、不正行為やスパム行為を目的としたツールは作成しない

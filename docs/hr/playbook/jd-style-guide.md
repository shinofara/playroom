# JD Style Guide（募集要項の書き方）

目的：募集要項を更新するたびに品質を再現できるようにする。

## 1. バイリンガル運用

- **日本語を正**とし、英語は日本語に忠実に追従させる
- 英語だけに新情報を追加しない（日本語にない魅力/条件/要件を入れない）
- 数字・強い主張は出典を確認（`company-facts.md` 参照）

## 2. 構成の型（推奨）

- 冒頭：ミッション / プロダクト / このポジションで実現したいこと（仕事の話に集中）
- ポジション特徴（候補者が一瞬で魅力を掴める箇条書き）
- （必要に応じて）チーム・開発組織について（言語/構成/海外比率など）
- 業務内容
- Seniorとしての期待（意思決定、設計、運用、横断推進）
- 技術的な面白さ・チャレンジ
- 要件（必須/歓迎）
- 条件（給与、働き方、VISA等）
- Tech Stack
- 選考プロセス

## 3. 今回の学び（再利用ルール）

- 「いま実現したいこと」に仕事と組織/働き方を混在させない
  - 組織情報（例：海外比率、公用語）は `Team & Engineering Communication` として独立させる
- 働き方が制度未整備の場合、言い切りを避けて `policies.md` の推奨文を使う
- AIツールは個人差があるため「利用は人により異なる」を必ず添える
- コーディングテストは「LLM利用が必要なレベル」として明示（候補者の認識齟齬を減らす）

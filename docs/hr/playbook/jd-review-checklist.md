# JD Review Checklist（募集要項レビュー観点）

## 事実・整合性
- [ ] DL数/ユーザー数、比率などの数字に出典がある（`company-facts.md`）
- [ ] 日本語と英語で内容が一致している（英語だけ増減していない）
- [ ] 働き方・VISA等の条件が他JDと矛盾していない（`policies.md`）

## 構成・読みやすさ
- [ ] 冒頭が「仕事/ミッション/実現したいこと」に集中している
- [ ] 組織/働き方の話は適切なセクションへ分離されている
- [ ] 箇条書きが過剰に長くなく、候補者が要点を掴める

## 要件の妥当性
- [ ] 必須/歓迎が現実的で、矛盾していない（年数、スコープ、言語要件）
- [ ] シニア期待値（意思決定、設計、運用、横断推進）が具体的

## 選考・期待値調整
- [ ] コーディングテストの有無・意図が明確（LLM利用前提）
- [ ] AIツール方針（利用は人による、良いものを試す）が誇張なく記載されている

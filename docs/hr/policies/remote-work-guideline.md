# リモートワーク・ガイドライン（原則出社／例外的リモート）

## 1. 目的（なぜ重要か）
- 当社は**オフィス出社を前提**としています。その上で、**会社全体のアウトプットを最大化するため**に、状況に応じてリモートワークを一部取り入れます。
- 「出社しなくていい」だけが目立って不公平感やモラル低下につながらないよう、リモートワークのルールを明確にします。
- リモート環境では「アウトプット」「気配」が不明瞭になり、面と向かっているより話しかけづらくなりがちです。**意図的にコミュニケーションしやすい状態を作る**ことが、信頼を築き、リモートがネガティブな影響を持たないために重要です。

## 2. 適用範囲
- 本ガイドラインは、当社の業務を遂行するメンバー（正社員・契約社員・業務委託・インターン等）を対象とします。
- ただし、雇用形態ごとの勤怠ルール・契約条件が優先される場合があります。

## 3. 基本方針（原則と例外）
- 働き方は**原則出社**です。
- ただし、明確な制度として固定されているわけではありません。家庭の事情や看病など、何らかの事情で出社が難しい日・期間がある場合は、**事前に相談のうえリモート勤務も可能**です（「この日だけ」といった単発の相談も含みます）。

### リモートが本当に必要か、まず考える
- リモートは便利な選択肢ですが、原則出社の前提の中では「本当にそれが必要か」を一度考えてください。
- 例：**宅配の受け取り**などは、土日受け取り等に切り替えられるなら、まずそちらを検討してください。
- 例：マンションの**火災報知器の点検**など、建物側が定めた日程から動かせないものは例外として発生し得ます。
- 「動かせる用事」と「動かせない用事」を切り分け、動かせる用事はできるだけ勤務時間外に寄せるのが基本です。

> 注：本ガイドラインは「出社／リモート」の取り扱いを定めます。
> **体調不良時の取り扱い（休むこと）**は別ドキュメント（`health-and-sick-leave-policy.md`）を参照してください。

## 4. リモートワークする人の心構え
- リモートワークは、オフィスにいるメンバーから**「いま働いているのか／進んでいるのか」が見えづらく、疑われやすい構造**があります（個人の善悪ではなく、環境の特性です）。
- 不必要に存在をアピールする必要はありませんが、**オフィスにいるメンバーが困らない状態を作ること**が大前提です。

### 大原則：見えない不安を作らない
- 「いまいるのか／いないのか」が分かる
- 「依頼した作業をやっているのか／やっていないのか」が分かる
- 「いつ返ってくるのか（返信・完了見込み）」が分かる

### リモートでは特に「ほうれん草」を強める
- オフラインでもほうれん草は重要ですが、リモートでは特に重要です。
- 小さな共有を増やすことで、周囲のブロック（待ち・不安・確認コスト）を減らします。

### 具体的なふるまい（例）
- 依頼を受けたら、まず**着手する／しない**を早めに返す（受領だけでもOK）
- 進捗が出ない作業ほど、**「何がボトルネックか／いつ次の更新が出せるか」**を短く共有する
- 離席や集中時間は、周囲が判断できる形で示す（詳細説明は不要）

## 5. リモート時のコミュニケーション（Must / Should / May）

### Must（必須）
- リモート勤務中は原則、**#voice-office のHuddleに参加し、カメラをオン**にする
  - オフィス側のカメラが切れていて「誰も見ていない」ように見える場合でも同様に守る
- **出退勤時**に、`#akasaka_office` でその旨を発信する
- **10分以上の離席**がある場合は、`#akasaka_office` で共有する（またはステータスで分かる状態にする）
- 緊急連絡が必要な場合に備え、上長・チームが連絡できる状態にする

### Should（推奨）
- 進捗や状況を、普段より細かく発信する（リモートでは情報が欠けやすいため）
- 雑談・軽い相談は `#random` も活用する（心理的安全性の維持）

### May（任意）
- 集中したいときは、ステータス（例：作業中）や「もくもく部屋」等を活用する

## 6. #voice-office / Slack Huddle の運用（常設の場）

### #voice-office（常設Huddle）のルール
- リモート勤務中は原則、**#voice-office のHuddleに参加し、カメラをオン**にします（MTG時に限りません）。
- オフィス側のカメラが切れていて「誰も見ていない」ように見える場合でも、**同じルールとして守ってください**。
  - リモート環境では「気配」や「声をかけてよいか」が伝わりにくいため、可視性の担保が重要です。
- 離席・集中などで一時的に応答しづらい場合は、ステータス等で周囲が判断できる状態にしてください。

### その他のHuddle（必要に応じて）
- #voice-office 以外のHuddleは、用途や参加者の合意に応じて運用します（必要に応じてカメラONを推奨）。

### 補足
- Slack の個人設定は、リモート運用がスムーズになるよう事前に調整しておくことを推奨します。
- クライアント（アプリ／OS）固有の挙動により、スリープ復帰で Leave 扱いになる等が起きる場合があります。参加中は接続状態を各自で確認してください。

## 7. 勤怠・労働時間（最低限）
- 勤務時間・休憩・中抜け・残業等は、雇用形態ごとのルールおよび勤怠制度に従います。
- リモートであっても労働時間管理が曖昧にならないよう、**始業・終業・離席**は分かる形にしてください。

## 8. 情報セキュリティ（最低限）
- 機密情報・個人情報を取り扱う際は、第三者に画面・音声が見聞きされない環境を確保してください。
- 紛失・誤送信・不正アクセスが疑われる場合は、速やかに上長および所定の窓口へ報告してください。

## 9. 例外・停止
- 業務上の必要（対面必須の会議、現物対応等）により、リモート中であっても出社を依頼する場合があります。
- 勤怠不備や情報管理上の懸念がある場合、リモート運用を停止・見直すことがあります。

# job-posting

募集要項を作成し、`docs/hr/career/` ディレクトリに保存します。

## 概要

このスキルは、新規ポジションの募集要項作成をサポートします。既存の募集要項フォーマットに沿って、一貫性のある高品質な募集要項を作成できます。

## 使用方法

### 1. 募集要項の作成依頼

ユーザーから以下のような情報を受け取った際にこのスキルを使用します：

- ポジション名（例: プロダクトエンジニア、プロダクトマネージャー）
- 雇用形態（正社員、業務委託など）
- 必要なスキル・経験の概要
- その他特記事項

### 2. ファイル命名規則

募集要項は以下の命名規則で保存します：

```
docs/hr/career/YYYY-MM-<ポジション名>.md
```

例:
- `docs/hr/career/2026-01-Senior-Product-Engineer.md`
- `docs/hr/career/2026-01-プロダクトエンジニアProduct Engineer.md`

### 3. 募集要項の構成

標準的な募集要項には以下のセクションを含めます：

#### 日本語版

1. **会社・プロダクト紹介**: 会社のミッション、プロダクトの概要
2. **仕事内容**: 具体的な業務内容（箇条書き）
3. **求める人物像**: どのような方を求めているか
4. **必要なスキル・経験**: 必須要件
5. **歓迎するスキル・経験**: あると望ましい要件
6. **給与・待遇**: 年収、ストックオプションなど
7. **働き方**: 勤務地、リモート可否など
8. **利用言語・プロトコル・ライブラリ・SaaS他**: 技術スタック

#### 英語版（必要に応じて）

1. **About the role**: 役割の概要
2. **Responsibilities**: 担当業務
3. **Who we're looking for**: 求める人物像
4. **Required Experience**: 必須経験
5. **Preferred Experience**: 歓迎経験
6. **Location/Work Style**: 勤務地・働き方
7. **Compensation**: 報酬
8. **Tech Stack**: 技術スタック

## 関連エージェント

このスキルは以下のエージェントと連携して使用できます：

- **engineer-job-post-writer**: エンジニア向け募集要項の作成に特化
- **recruitment-spec-orchestrator**: 複数の専門エージェントを調整して募集要項を完成

## ワークフロー

1. **情報収集**: ユーザーからポジションに関する情報を収集
2. **テンプレート選択**: 類似の既存募集要項を参照
3. **下書き作成**: 募集要項の下書きを作成
4. **レビュー**: ユーザーと内容を確認
5. **ファイル保存**: `docs/hr/career/` に適切なファイル名で保存

## 注意事項

- 給与情報は会社のポリシーに従って記載
- 技術スタックは最新の状態を反映
- 日本語・英語の両方で作成する場合は、内容の一貫性を保つ
- 必須/歓迎スキルは現実的かつ具体的に記載
